## main.py

- Connects to PostgreSQL on startup using asyncpg
- Loads all plugins dynamically from plugins/ folder
- Registers global check: if guild not whitelisted silently ignore all commands
- On startup checks all current guilds against whitelist, leaves non-whitelisted
- On on_guild_join checks whitelist, leaves if not whitelisted, DMs server owner and notifies bot owner
- on_message: resolves aliases from YAML before command dispatch
- Starts all background tasks on ready
- Re-registers all persistent views (reaction roles, tickets) on startup using stored message IDs

---

## config_loader.py

- Loads guild YAML config from guild_configs table
- Caches configs in memory with 60 second TTL
- Returns parsed dict
- Falls back to default config if none set

---

## duration_parser.py

Parse duration strings: 10m, 1h, 1d, 1w, 2d12h, perm
MAX_DURATION_DAYS = 180
Cap all durations at 180 days.
If capped, note it in the response.
Returns (timedelta or None, human_readable_string, capped bool)

## message_formatter.py

Every bot response goes through this.
Three formats supported:
1. Plain string
2. Embed only
3. Content + embed combo

Default is always plain text.
Embeds opt-in via YAML only.

Template variables supported everywhere:
{user} {user.mention} {user.id} {user.name} {user.avatar} {user.created_at} {user.joined_at}
{mod} {mod.mention} {mod.id} {mod.name}
{server} {server.id} {server.icon} {server.member_count}
{reason} {duration} {case_id} {action} {channel} {channel.mention} {count} {expires_at}
{new_reason} {new_duration} {case_type} {trigger} {rule}
{timestamp} {timestamp.date} {timestamp.time}
{success_count} {fail_count} {total}
{reminder_message} {reminder_set_at} {ordinal}

Missing variables fall back to N/A.
Colors support hex strings "#FF0000".

Default punishment channel reply:
"{user} has been {action} | Case: {case_id}"

Default punishment DM:
"You have been {action} in {server} for the following reason: {reason}"

Ban and mute DMs:
"You have been {action} in {server} for the following reason: {reason} | Duration: {duration}"

## Whitelist System

Only whitelisted servers can use the bot.
On startup: check all guilds, leave any not in whitelisted_guilds.
On guild join: check whitelist, leave if not whitelisted.
Non-whitelisted servers: bot silently ignores all commands.
Bot owner bypasses everything always.

## Config Access System

Three tiers:
1. BOT_OWNER_ID — full access everything
2. guild_owner_id from whitelisted_guilds — setconfig/getconfig/resetconfig/configaccess for their server
3. config_access table users — setconfig/getconfig for their server only

Commands:
- !setconfig — paste YAML block, bot validates and saves
- !getconfig — DMs current raw YAML
- !resetconfig — resets to default YAML
- !configaccess add @user — whitelisted owner only
- !configaccess remove @user — whitelisted owner only
- !configaccess list — whitelisted owner only

- Creates case where applicable
- DMs user if dm_on_action true

- Logs to logging channel
- Triggers escalation check after execution
- Every response fully customizable in YAML
- Every DM fully customizable in YAML
- Every error fully customizable in YAML
- Every log message fully customizable in YAML
- Force commands take raw user ID, work without user in server, DM failures silent

## plugins/escalation.py

Two systems. All responses fully customizable in YAML.

Manual escalation:
Fires after warn/mute/kick.
Counts existing cases of tracked_type.
If count matches threshold: apply escalation action.
Creates new case, DMs user, logs.
tracked_type "any" counts all types.

Automod escalation:
In memory violation counts per guild per user per rule.
Reset after reset_after_hours of inactivity.
Find highest threshold where count >= threshold count.
Apply action, create case, DM user, log.

Expose:
- await check_manual_escalation(ctx, guild_config, user, action_type)
- await check_automod_escalation(guild, guild_config, member, rule_name, channel)

          action: kick

          action: ban
          duration: "perm"
          reason: "Escalation: 10 warnings"
        - tracked_type: "mute"
          count: 3
          action: kick
          reason: "Escalation: 3 mutes"

          count: 5
